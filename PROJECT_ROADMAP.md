## P1 · 익명 감정 피드 알림 기능
- **status**: backlog
- **milestone**: MVP 이후 커뮤니티 기능 단계
- **okr**: DAU 증가와 커뮤니티 피드 사용률 향상
- **why**: 익명 피드를 향후 추가할 계획이므로, 사용자 재방문과 커뮤니티 활성화를 위한 알림 시스템이 선제적으로 필요한 시점임.
- **success**: 사용자가 포모 앱에서 자신이 참여한 감정 투표나 피드 관련 알림 메시지를 수신하는 화면이 확인됨. 알림 설정 가능.
- **scope**: 감정 투표, 피드 활동에 기반한 맞춤형 알림 발송 기능 및 알림 설정 UI 개발
- **breakdown**: 기획: 알림 메시지 종류 및 조건 정의 / 백엔드: 감정 투표, 피드 활동 이벤트 기반 알림 발송 로직 작성 + 알림 DB 설계 / 프론트·UX: 알림 설정 UI 구현 (알림 ON/OFF 체크박스 등) 및 알림 페이지 화면 설계 / 품질: iOS, Android, 웹 PUSH 알림 테스트

## P2 · 나의 감정 패턴 분석(진화적 인사이트)
- **status**: backlog
- **milestone**: N/A
- **okr**: 사용자 경험 차별화를 통한 리텐션 향상
- **why**: 개인화된 감정 경험 제공이 제품 가치를 높이는 핵심 요소며, 개인 데이터 가공 및 분석은 '나만의 데이터 통찰' 욕구를 충족시킬 수 있음.
- **success**: 사용자가 자신의 감정 변화 패턴을 시계열 데이터로 시각화하여 확인 가능.
- **scope**: 감정 입력 데이터를 활용한 사용자 맞춤 분석 시스템 개발 및 시각화를 위한 편집 대화형 컴포넌트 추가
- **breakdown**: 기획: 감정 패턴 분석 로직 설계 및 UX 흐름 정의 / 백엔드: 감정 데이터를 시간 단위로 축적 및 분석하는 API 제작 / 프론트·UX: '나의 감정 패턴' 대화형 그래프 컴포넌트 개발 및 감정 연속 데이터를 표시하는 인터페이스 설계 / 품질: 데이터 축적 및 분석 API 성능 검증 및 패턴 보기 UI 품질 테스트

## P3 · 포인트 소비 메커니즘 및 리워드 시스템
- **status**: backlog
- **milestone**: 적립 로직 연계 확장
- **okr**: 재방문 증가 및 사용자 참여도 향상
- **why**: 포인트 적립 후 소비 로직이 현재 비어 있음. 사용자는 포인트를 활용할 수 있는 명확한 동기 필요.
- **success**: 사용자가 획득한 포인트를 감정 관련 디지털 아이템 구매 또는 특별 기능(예: 고급 감정 통찰)에 사용할 수 있음.
- **scope**: 포인트 사용 규칙 및 거래 내역 API 생성, 리워드 구매 가능한 상점 화면 및 상호작용 개발
- **breakdown**: 기획: 포인트 소비 로직 및 리워드 아이템 설계 / 백엔드: 포인트 차감 및 리워드 지급 API 개발 / 프론트·UX: 사용자 상점 화면 및 포인트 사용 흐름 설계 / 품질: 포인트 거래 및 리워드 지급 기능 검증

## P4 · 커뮤니티 밈 생성 기능
- **status**: backlog
- **milestone**: 밈 활성화 프로젝트
- **okr**: 사용자 콘텐츠 공유 증대로 자연 유입 증가
- **why**: 투자자 감정을 콘텐츠화하여 확산시키고, 사용자 참여를 유도하는 수단으로 활용 가능. 감정 시각화를 더 큰 시장으로 확대할 기회.
- **success**: 사용자가 자신의 감정을 기반으로 맞춤 밈을 생성하고, 이를 다운로드 및 공유 가능.
- **scope**: 감정 밈 템플릿 제작, 사용자 감정을 반영한 밈 생성기 개발, 다운로드 및 공유 지원
- **breakdown**: 기획: 감정 밈 템플릿 설계 및 콘텐츠 규격 정의 / 백엔드: 밈 생성 서비스/API 개발 및 데이터 저장 로직 구현 / 프론트·UX: 밈 생성기 인터페이스 및 다운로드/공유 기능 구현 / 품질: 밈 생성 퀄리티 및 인터페이스 검증

## 진행 원칙
- **활성은 동시 1개.** 새 프로젝트 선택 시 이전 active 는 backlog 로 강등(완료는 명시적 done 처리).
- 활성 프로젝트가 있으면 직군은 그 프로젝트 이슈만 쪼개고 격파한다. 산발적 일일 제안 금지.
- 활성 프로젝트가 없으면 daily 리포트는 백로그를 surfacing 하고 오너의 선택을 기다린다.
- 프로젝트 완료 → retro → 다음 후보 surfacing.
