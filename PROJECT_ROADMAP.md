## P1 · 감정 캘린더 개발
- **status**: backlog
- **milestone**: M2
- **okr**: O3
- **why**: 단 하나의 순간을 매일의 습관으로 확장해, 변동성 없는 날에도 사용자들이 포모를 지속적으로 찾도록 하는 이유를 제공함으로써 장기적인 사용자 리텐션을 강화한다.
- **success**: 감정 캘린더를 통해 사용자가 한 달 간의 감정 기록을 확인할 수 있고, 빈 칸을 채우는 행위와 스트릭 기능이 도입되어 잔잔한 날에도 사용자가 앱을 자연스럽게 열게 됨.
- **scope**: 감정 캘린더 UI/UX 설계, 스트릭 기능 구현, 잔잔한 날을 위한 포모 동작과 메시지 개발, Index 흐름 시각화 추가.

## P2 · 혼자가 아님을 체감하는 기능 확장
- **status**: backlog
- **milestone**: M3
- **okr**: O2
- **why**: 집단적 감정을 통해 사용자들이 '혼자가 아니다'라는 위로를 받을 수 있도록 하고, Index의 신뢰성을 바탕으로 정직한 숫자를 유지하여 초기 사용자 경험을 강화한다.
- **success**: 홈 화면에서 참여자 수와 감정 현황이 정확히 집계되고, 고래 트래킹 카드가 초기 사용자들에게 데이터를 기반으로 안도감을 제공하며, 정직한 숫자로 제품의 신뢰성을 확보함.
- **scope**: 집단 감정 데이터 통합, 정확한 참여자 집계 및 노출 구현, 고래 트래킹 카드 개발 및 시각화, 데이터 폴백 시스템 강화를 포함.

## P3 · 불안 해소 여정의 완성
- **status**: backlog
- **milestone**: M4
- **okr**: O1, O3
- **why**: 제품이 감정을 표현하고 회복하는 사이클을 완결 지을 수 있도록 하여 사용자가 불안을 해소하고 이탈 없이 지속적으로 앱에 머물도록 함.
- **success**: 사용자가 물렸을 때 다른 앱으로 이탈하지 않고, 감정 표현과 회복 사이클이 제품 내에서 완결되어 재방문율 및 감정 안정 경험이 강화됨.
- **scope**: 감정 선택 후 보다 깊이 있는 회복 과정 설계, 추가 솔직한 메시지와 포모 표정의 섬세함 강화, 감정 사이클 완결 UX 개발.

## 진행 원칙
- **활성은 동시 1개.** 새 프로젝트 선택 시 이전 active 는 backlog 로 강등(완료는 명시적 done 처리).
- 활성 프로젝트가 있으면 직군은 그 프로젝트 이슈만 쪼개고 격파한다. 산발적 일일 제안 금지.
- 활성 프로젝트가 없으면 daily 리포트는 백로그를 surfacing 하고 오너의 선택을 기다린다.
- 프로젝트 완료 → retro → 다음 후보 surfacing.
