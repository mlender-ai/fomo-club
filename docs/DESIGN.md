# FOMO Club — DESIGN.md

> 디자인 시스템 **단일 출처**. Claude Code(디자인·UX)와 Codex(데이터 렌더 시)가 모두 이 문서를 참조한다. 모든 디자인 핸드오프는 여기서 파생하며, 토큰이 충돌하면 **이 문서가 이긴다.**
> 컨셉 한 줄: **살아있는 터미널 × 클린 시트** — 발견은 네온/픽셀(에너지), 근거는 클린(신뢰), 둘을 픽셀 디졸브로 잇는다.
> 테마: **다크 퍼스트, 단일 테마(v1).** 라이트 모드는 범위 밖(미래).

---

## 1. Identity & Principles

- **이중 모드:** 발견 모드(홈·스와이프·카드 앞면) = 살아있는 터미널. 근거 모드(딥프로필·출처·재무) = 클린 시트. "겉은 가볍게, 속은 단단하게"의 시각 구현.
- **브랜드 대비:** 머니터링 = 라이트·블루·빽빽한 점수·은행 도구. 우리 = 다크·네온/오렌지·픽셀·alive. 의도적으로 다른 종(種).
- **제품 원칙이 시각을 구속한다(절대):**
  - 색이 매수/매도를 인코딩하지 않는다. 브랜드색 = 열기/발견, 등락색 = 데이터 전용(봉인).
  - 무판정·차분: 근거 모드는 절대 화려하게 만들지 않는다.
  - 상태는 색만으로 전달 금지 — 항상 텍스트 라벨 동반(접근성·무판정).

## 2. Color

다크 퍼스트. 라이트니스 단계로 표면 구분(소프트 섀도 아님).

**Base / Surface**
| token | hex | 용도 |
|---|---|---|
| `--canvas` | `#0B0B0C` | 최하단 배경 |
| `--surface` | `#141416` | 기본 표면 |
| `--surface-raised` | `#1A1A1D` | 카드 |
| `--surface-overlay` | `#202024` | 모달·시트 |
| `--hairline` | `rgba(255,255,255,.08)` | 기본 보더 |
| `--hairline-strong` | `rgba(255,255,255,.14)` | 강조 보더 |

**Text**
| `--text-primary` `#F2F2F0` | `--text-secondary` `#8A8A86` | `--text-tertiary` `#5A5A57` |

**브랜드 액센트 — 역할로 인코딩 ("좋다/사라" 아님)**
| token | hex | 역할 |
|---|---|---|
| `--orange-500` | `#FF5A1F`* | **주목 열기/강도.** 🔥 한복판, 포모 강도 미터, 픽셀 스파크라인 |
| `--orange-600` / `--orange-dim` | `#E0440F` / `#5A2A14` | 강조 / 저강도 트랙 |
| `--neon-500` | `#D8FF3A`* | **발견·인터랙션·💎.** 1차 CTA(관심), 활성/선택, 💎 특별, live 펄스 |
| `--neon-600` / `--neon-dim` | `#B6E000` / `#2E3A0A` | hover / 저강도 |

*\*오렌지는 현재 브랜드 토큰이 있으면 그걸로 교체. 네온은 시안 보고 튜닝.*

**봉인색 — 등락 데이터 전용 (브랜드로 절대 사용 금지)**
| `--up` `#FF4D4D` (상승, KR red=up) | `--down` `#4D8DFF` (하락) | `--flat` `#8A8A86` |
가격·%·변동에만. 오렌지/네온을 등락에 쓰면 매수신호 색코딩 = 원칙 위반.

**근거 보조 (절제)**
- 그린/레드플래그 = 라벨 칩(아이콘+텍스트). 색은 보조 큐만: green `#3FB984` / red `#E0A82E` 헤어라인 수준. **fill 판정 금지**, 등락색과 구분, 텍스트 항상 동반.
- confidence = 중립 도트 등급(색 verdict 아님).

## 3. Typography

| family | 토큰 | 폰트 (확정) | 용도 | 금지 |
|---|---|---|---|---|
| 픽셀·모노 (라틴) | `--font-display` / `--font-mono` | **Departure Mono** (SIL OFL) | 포모 점수·큰 통계·티커(디스플레이) + 작은 수치·%·가격·타임스탬프(데이터) + 픽셀 스파크라인·박스드로잉 UI | **한글 금지** |
| 한글·문장 | `--font-sans` | **Pretendard** (SIL OFL) | 모든 한글·헤드라인·본문·라벨·버튼 | — |

- **확정:** Departure Mono(라틴 숫자·데이터·픽셀 UI) + Pretendard(한글·문장). 둘 다 **SIL OFL = 상업용·웹·앱 임베딩 무료, 로열티 0.** Departure Mono가 디스플레이·데이터를 한 패밀리로 커버 → 스택 단순화(파일·일관성 유리). 픽셀 퍼펙트는 11px 배수에서.
- (옵션) **Silkscreen**(OFL) — 더 강한 8비트가 필요한 브랜드 모먼트(로고·스플래시)만.
- **OFL 의무 3:** ①폰트 단독 판매 금지(앱 번들은 OK) ②배포 시 라이선스 텍스트 동봉 ③예약명으로 수정본 배포 금지. 일반 앱·웹 사용은 전부 허용.

**Scale (mobile):** display-xl 56 / display-l 40 / h1 24 / h2 20 / h3 17 / body 15 / caption 13 / micro 11. line-height: 디스플레이 1.0~1.1, 한글 본문 1.5~1.6.
**Weight:** Pretendard 400(본문)·500(라벨)·700(헤드라인). 픽셀 폰트 단일. **600/heavy 남용 금지.**
**규칙:** 숫자=픽셀(라틴, 큰 것만) / 작은 숫자·%=모노 / 한글·문장=Pretendard. 등락 %는 모노 + 등락색.

## 4. Spacing & Layout

- Base 4px. Scale: 4·8·12·16·20·24·32·40·48.
- Mobile 거터 20. 카드 패딩 16~20. 컴포넌트 내부 갭 8·12.
- 모바일 퍼스트 ~380. 세이프에어리어 준수. 스와이프 덱은 풀블리드.

## 5. Shape & Border

- Radius: `--r-sm 8` / `--r-md 12`(기본) / `--r-lg 16`(카드) / `--r-pill 999`.
- 카드는 라운드(Tinder의 부드러움). **픽셀/터미널 패널**(스파크라인 프레임·미터)은 radius 0 허용 — 디지털 질감.
- 보더: 1px `--hairline`. 싱글사이드 보더에 라운드 금지.

## 6. Elevation & Texture

- **플랫 다크.** 표면 구분은 라이트니스 단계(canvas<surface<raised<overlay) + 헤어라인. **소프트 드롭섀도 금지.**
- 허용: 스와이프 활성 시 **기능적 네온 엣지 글로우**(정적 요소엔 금지). 발견 모드 한정 도트/스캔라인 텍스처 ≤4% opacity.
- 금지: 그라데이션 워시, 정적 글로우, 블러 섀도, 근거 모드의 네온.

## 7. Motion

- **제스처:** 드래그 카드 — 우=관심 / 좌=패스 / 상=슈퍼관심. 스프링 물리.
- **피드백:** 관심 → 네온 엣지 펄스 150ms + 햅틱 · 패스 → 디새추레이트 + 픽셀 디졸브 200ms · 슈퍼관심 → 네온 픽셀 버스트 300ms.
- **모드 전환:** 더보기 → 플립 + 픽셀화→resolve 350ms(발견↔근거 유일한 다리).
- **Duration/Easing:** micro 120 / standard 200 / expressive 320. ease `cubic-bezier(.2,0,0,1)`. 디졸브·미터는 `steps()`로 디지털 느낌.
- **60fps 필수**, 스와이프 잰크 금지. `prefers-reduced-motion` 존중(디졸브→페이드, 버스트 off).

## 8. Motifs — 데이터 비주얼 프리미티브 (FOMO 시그니처)

- **픽셀 스파크라인:** 매끈 선 아님 — 이산 픽셀 컬럼. 오렌지 단색(가격 *형태*, 판정 아님). 모든 증권앱과 차별.
- **포모 강도 미터:** 10 도트 세그먼트, 점수만큼 오렌지 fill, 나머지 dim. 옆에 디스플레이 폰트 숫자.
- **도트-그리드 히트맵:** 발굴 성적표/활동(GitHub식, 네온 4단계). 후속.
- **로딩 시머:** 픽셀 블록 사이클(로딩 진행 작업과 연결).

## 9. Components

**Stock Card (발견 모드)** — `--surface-raised`, `--r-lg`, 패딩 16~20, 스와이프
1. 정체성: 로고 40px 원 + 종목(Pretendard 17/700) + 코스닥·시총 N위(mono/secondary)
2. 포모 배지: "포모" micro + 숫자(display) + 상태 라벨. 숫자/액센트 색 = 상태별(🔥 오렌지 / 💎 네온 / 데우는중 orange-dim / 조용·식는중 secondary). **상태 텍스트 항상.**
3. 테마 칩 · 4. 헤드라인(Pretendard 15) · 5. 픽셀 스파크라인 · 6. baseline(mono, 하단 작게)

**State chips:** 🔥 지금 한복판 / 💎 오기 직전 / 데우는 중 / 조용 / 식는 중. 글리프+텍스트(색만으로 금지).

**Buttons:** 관심 = 네온 pill(bg `--neon-500`, text `#0B0B0C`/700) · 패스 = 고스트 원(헤어라인, ti-x) · 슈퍼관심 = 네온 아웃라인 버스트. **탭 타깃 ≥44px.**

**Deep Profile (근거 모드 / 클린 시트)** — `--canvas`, 여백↑, 중립 팔레트, 네온 최소
- 순서: 포모 상태 히어로 → 왜 → 돈 → 미래 → 차트 뒷받침 → 회사 한 줄
- Evidence row: 라벨(secondary) / 값 / 출처·시점(micro mono) / confidence 도트(중립)
- 그린/레드플래그 칩: 아이콘+텍스트, 헤어라인 색 보조, fill 판정 금지

## 10. Voice & Tone (UI 카피) & Icon

- 해요체. 무판정(매수·매도·목표가·상승확률 금지). 시점·출처·confidence 항상. "추천" 대신 "취향 매칭·발견·판단 재료".
- 아이콘: 라인 세트(Tabler outline 류) 일관 스트로크. 픽셀 커스텀 글리프는 브랜드 모먼트(로고·상태 마크)만.

## 11. Mode Reference

| | 발견 모드 | 근거 모드 |
|---|---|---|
| 배경 | canvas, 텍스처 ≤4% | canvas, 텍스처 없음 |
| 색 | 오렌지/네온 액센트 | 중립 + 최소 액센트 |
| 타이포 | 픽셀 숫자 강조 | Pretendard 중심, 숫자 차분 |
| 모션 | 스와이프·디졸브·버스트 | 정적·전환 절제 |
| 느낌 | alive·FOMO | 신뢰·차분 |

## 12. Do / Don't

**Do:** 네온은 날카로운 액센트 · 상태는 색+텍스트 · 등락색 봉인 · 근거 모드 차분 · 60fps · WCAG 대비 확인.
**Don't:** 매수/매도 색코딩 · 한글 픽셀폰트 · 정적 글로우/그라데이션 워시 · 근거 모드 네온화 · 미감 위해 무판정·차분 원칙 훼손.

## 13. Governance

- 이 DESIGN.md = 단일 출처. 핸드오프(`DESIGN_LANGUAGE_HANDOFF` 포함)는 *적용 순서·화면별 지시*로 격하, 토큰은 여기서 가져감. 충돌 시 이 문서 우선.
- 오너: 디자인 시스템 = Claude Code / 데이터 렌더 준수 = Codex / 의사결정 = 광혁·Claude(기획).
- 변경 절차: 토큰·컴포넌트 변경은 이 문서부터 → 파생 핸드오프 갱신.

## 구현 상태 (코드 정합 — 이 섹션은 적용 진척 추적용)

- ✅ **토큰 머신리더블 미러:** `design/tokens.json`(DTCG) 에 위 색/타이포/스페이싱/radius 반영. 감정색(레거시)은 `@fomo/core` EMOTION_COLORS 와 드리프트 테스트로 고정.
- ✅ **Tailwind 토큰(fomo-web):** `tailwind.config.ts` 에 DESIGN.md 토큰을 *추가*(canvas·surface-raised·surface-overlay·hairline-strong·text-*·orange*·neon*·up/down/flat·radius). 기존 토큰(surface/hairline/whiteout/muted)은 점진 마이그레이션 위해 보존.
- ⏳ **폰트 스왑(Galmuri → Departure Mono):** Departure Mono 는 CDN 부재(OFL zip) — 폰트 에셋 호스팅/`@font-face` 셋업이 선행 필요. 그전까지 Galmuri 유지(픽셀 스택 폴백에 Departure Mono 포함).
- ⏳ **컴포넌트 적용(카드·딥프로필·버튼·픽셀 스파크라인·포모 미터):** §13 대로 화면별 파생 핸드오프로 진행. 현 카드/상세는 기존 토큰 사용 중 → 핸드오프 시 DESIGN.md 토큰으로 전환.
- 봉인색(up/down) 원칙: 현재 일부 컴포넌트가 등락에 코랄/블루 하드코딩 — 마이그레이션 핸드오프에서 `--up`/`--down` 토큰으로 정리.
