## 0. 왜 이 문서가 필요한가 (현 상태의 사실)

제품 정의가 **3겹으로 어긋나** 있고, 에이전트 군단이 이미 폐기된 모델을 최적화 중이다.

| 문서 | 주장하는 제품 | 현실 |
|---|---|---|
| `FOMO_CLUB.md` | 홈 = FOMO Index + **감정 투표** + 집단 통계 | 폐기됨 |
| `PIVOT_FEED_FIRST.md` | 오늘/피드 탭 + **감정 카테고리** 뉴스 카드 | 폐기됨 |
| **실제 앱(스크린샷)** | **카드/히스토리 탭 + 키워드 스와이프 카드 + 키워드별 포모 점수** | 현재 |

`AGENT_NORTH_STAR.md`는 아직 "감정 투표 참여"를 측정 지표로 둔다 → **조직이 떠난 좌표로 일하는 상태.** 이 문서로 좌표를 통일한다.

## 1. 현재 제품 정의 (단일 진실)

### 한 줄 정의
**매일 와서 "오늘 다들 뭐에 쏠렸는지"를 보고, "안 따라가도 괜찮아"를 듣고 가는 곳.**

### 코어 밸류
토스가 "얼마 벌었나"를 보여준다면, 포모클럽은 "안 사도 괜찮아"를 매일 말해준다.
정보 앱이 아니라 **감정 진정 앱**이다. 시장을 분석하지 않고, 정보를 감정으로 번역해 안심시킨다.

### 코어 루프 (제품의 차별점)
```
오늘 쏠린 키워드를 보여준다 (포모 유발)
        ↓
틴더처럼 슥슥 넘긴다 (수동 소비, 액션 강요 X)
        ↓
각 카드가 즉시 진정시킨다 (균형추: "기회는 또 와")
        ↓
넘긴 데이터가 쌓인다 (개인화의 연료)
```
뜨거운 걸 보여주고 → 즉시 진정시키는 이 루프를 하는 앱은 없다.

### 마스코트 포모의 새 위치
- 과거 정의("화면의 주인공은 표정 짓는 포모")는 키워드 카드 피벗으로 **강등됐다.**
- 진짜 주인공은 **친구의 목소리(코멘트)**다. 차별점이 카피에 있다.
- 포모의 얼굴은 각 키워드 카드의 **미니 감정 신호**로 재배치한다(키워드 포모 점수 → 작은 포모 표정). 마스코트를 새 모델에 다시 연결.

## 2. 폐기 / 보관 (코드 정리 — 파일 경로)

### 즉시 잘라낼 것 (legacy 브랜치 보관 후 main에서 제거)
타로는 초기 컨셉의 잔재다. "보존"은 공짜가 아니다(에이전트 인지 부하 + 매번 예외 명시 + 과거 운영 URL에도 남았던 레거시 이름). git 히스토리가 보존하므로 main에 둘 이유 없다.

- `packages/tarot-core/`
- `apps/tarot-mobile/`
- `apps/web/app/api/tarot/`
- `apps/web/lib/tarot/`
- `docs/사주팔자_투자체질_기획서.md`
- 패키지/디렉토리 리네이밍(taro → fomo)도 이 정리 라운드에 포함.

### feature flag로 보관 (삭제 X, 코드·DB 유지)
감정 투표 홈 모델. PIVOT_FEED_FIRST.md의 플래그 정책 유지.
- `FEATURE_EMOTION_VOTE`, `FEATURE_EMOTION_JOURNAL`, `FEATURE_EMOTION_CALENDAR`, `FEATURE_HISTORY_TAB`
- 위치: `packages/fomo-core/src/features.ts`
- 관련 컴포넌트(`EmotionGate`, `VoiceFeed`, `EmotionCalendar`)와 `/api/fomo/emotions/*`는 보관. 나중에 니즈 생기면 복원.

### 앱(RN) 보류
- `apps/fomo-club`(Expo)는 보류. **웹 단일 초점.**
- 이유: 출시 전 1인 빌더가 SNS 1,000팔로워를 쫓는 단계. 웹은 공유 링크·설치 마찰 0으로 SNS 깔때기에 맞는다. RN+웹 동시 유지는 표면 2배.
- 웹이 핵심 가설(D1/D7 재방문)을 검증한 뒤 앱 재개.

## 4. 새 North Star (이번 분기)

### 테마
**"화면의 모든 숫자를 진짜로 + 키워드 엔진 + 웹 단일 초점"**

### 한 줄 메시지
이번 분기 모든 작업은 ① 키워드 카드를 mock에서 실데이터로, ② 타로 잘라내고 문서·좌표 통일, ③ 웹에서 매일 신선한 인사이트가 도는 경험. 세 축에만 집중한다. 기능 추가·앱(RN)·신규 카테고리는 자동 폐기.

### 절대 제안 금지 (자동 폐기)
- **가짜/임의 숫자**: 실제 집계 없는 키워드 점수, 폴백을 진짜처럼 노출.
- **투자 조언·예측**: 종목 추천, 매수·매도·목표가, 미래 단정.
- **앱(RN) 작업**: 웹 검증 전까지.
- **감정 투표 홈 복원**: flag OFF 유지.
- **신규 카테고리 확장**: Now는 코인+주식 테마만. 부동산·환율은 Later.
- **기능 비대화**: MLP는 기능 수가 아니라 "매일 진짜 인사이트"의 신선도·정직함.

## 5. 측정 지표 (죽은 지표 교체)

> 옛 지표(감정 투표 참여, 4 Heat 폴백)는 폐기 모델의 것. 새 모델의 지표로 교체.

| 지표 | 현재 | 목표 |
|---|---|---|
| 키워드 카드 실데이터 비율 (vs mock) | 0% (전부 mock) | 100% |
| 키워드 스냅샷 일일 연속성 | 없음 | 매일 1건, 누락 0 |
| D1 / D7 재방문 (익명 세션) | 미측정 | 측정 시작 + 기준선 |
| 세션당 스와이프한 카드 수 | 미측정 | 측정 시작 |
| 깊이 페이지 열람률 (카드 → depth) | 미측정 | 측정 시작 |
| 개인화 신호 수집량 (관심/덜관심 + 본 키워드) | 로컬만 | 서버 집계 시작 |
| confidence 노출 정확성 (폴백을 폴백으로) | 0% (숨김) | 100% |

> 북극성 후보: **D7 재방문**(습관 형성의 증거). 보조: 세션당 카드 수·깊이 열람률(콘텐츠가 진짜 가치 있는지).

## 6. 방향 시퀀스 (Now / Next / Later)

- **Now (루프 검증)**: 코인+주식 테마 하나를 미친 듯이 잘. 매일 신선한 실데이터 키워드 카드. 지표 = D1/D7 재방문.
- **Next (개인화 쐐기)**: 스와이프(관심/덜관심) + 본 히스토리(`keywordHistory`)로 덱 정렬·코멘트 톤 개인화. "너는 코인엔 안 흔들리는데 반도체엔 약하더라" = 개인 포모 프로파일 = 해자.
- **Later (확장)**: 부동산·환율·원자재. 더 멀리는 투자 밖 FOMO(영끌·취업). 브랜드가 "투자 포모"에서 "모든 비교-불안의 진정제"로.

## 7. BM 확장 (조언을 팔지 않는다는 전제 위에서)

내러티브: **무료 일일 안티-포모 피드 → 습관 + 감정적 신뢰 + 관심 데이터 → 두 자산이 복리로: (1) 관계, (2) 지수.**

1. **개인화 프리미엄(1순위)**: 무료=오늘의 키워드 카드. 유료=개인 포모 프로파일, 테마 깊이 보기, 과거 패턴, 알림("네가 약한 반도체가 또 떴어 — 근데 진정해"). 조언이 아니라 관계·개인화를 판다. 정체성과 완벽 정합.
2. **FOMO Index 라이선싱(장기, B2B2C)**: 집계된 군중 쏠림 지수를 미디어·증권사에 라이선스(CNN Fear & Greed 모델). 단 지수가 진짜·신뢰받을 때만 — 그래서 "숫자 진짜화"가 BM의 전제.
3. **타로 부활(선택·재미)**: 죽이지 말고 서랍에. 감정 데이터 쌓이면 "오늘 나의 투자 감정 리딩" 엔터테인먼트 유료 레이어로 부활 가능. 조언 아니라 재미라 정체성과 안 부딪침.
