## 1. 한마디 정의

### 내부 좌표 (팀·문서·에이전트용)
**투자 콘텐츠의 취향 기반 알고리즘 피드.** 릴스가 내 취향대로 요리/주식 채널을 흘려주듯, 포모클럽은 내 취향대로 시황·종목·테마 카드를 흘려준다. 매일 새로운 인사이트를 스와이프로 소비하고, 그 과정에서 "따라가도 될지" 판단하게 돕는다.

### 외부 한 줄 (SNS·랜딩·피칭용)
**"종목을 분석하는 앱은 많아요. 포모클럽은 그 종목에 흔들리는 당신을 분석해요."**

### 왜 이 정의인가 (진화 경위)
- v1은 "흔들리는 나를 판단하게 돕는 앱"이었다. 맞지만 진입 방식이 빠져 있었다.
- 진짜 메커니즘은 **취향 알고리즘 피드**다. 스와이프(넘김=관심없음 / 멈춤=관심 / 깊이봄=강한관심)가 곧 취향 학습 신호다. 틴더가 스와이프로 이성 취향을 배우듯, 포모클럽은 스와이프로 투자 취향을 배운다.
- **이 자리는 비어 있다.** 토스는 "전체 인기 종목"을 모두에게 똑같이 보여준다(랭킹, 알고리즘 아님). 유튜브·릴스는 취향 알고리즘이 있지만 영상이지 투자 정보가 아니다. "투자 정보를 취향 기반으로 큐레이션한 피드"는 아무도 안 한다. → 검색(토스도 함)보다 강한 차별점.

### 발견형 vs 검색형 — 본체와 보조
- **발견형(본체)**: 평소 앱을 켜면 내 취향 시황·종목 카드가 피드로 흐른다. 매일 여는 습관 = 리텐션 바닥.
- **검색형(보조)**: 릴스/커뮤니티에서 새 종목에 꽂혔을 때만 발동. "서비스나우 검색 → 한 번 더 더블체크". 니즈가 있을 때만.
- 둘은 경쟁이 아니라 레이어다. 발견형이 리텐션을 만들고, 그 위에서 검색형이 결정적 순간(결제 의향 최고점)을 잡는다. 목적지는 통합.

### 정의의 두 축
- 매일 오는 이유(훅) = 취향대로 흐르는 신선한 피드. 정보는 입장권.
- 차별점(해자·BM) = **개인화 취향 매칭** — "네 취향을 학습해 너만의 피드를 매칭"하는 것. "정보를 캐치"로 정의하면 레드오션(토스)에 빨려든다 — 정보로는 못 이긴다. **취향 알고리즘 매칭 + 쉬운 번역**이 우리 자리(= 증권시장의 틴더). 판단·결정은 유저 몫.

## 2. 포지셔닝 — 증권시장의 틴더 (취향 매칭)

> **한 문장 해자: 토스·카카오는 "시장이 어떤지"를 모두에게 똑같이 보여준다. 포모클럽은 "네 투자 취향"을 학습해 "너만의 피드"를 매칭한다. = 증권시장의 틴더.**

두 축: **모두 같은 화면 ↔ 각자 다른 화면** / 어렵게(원본) ↔ 쉽게(번역).

| | 모두 같은 화면(랭킹) | 각자 다른 화면(취향 매칭) |
|---|---|---|
| **어렵게(원본 데이터)** | 증권사 HTS, 돈이돼지 RSI·수급 | (없음) |
| **쉽게(번역)** | 토스·카카오 AI 요약 (전체 인기) | **포모클럽 (내 취향 × 쉬운 번역)** |

- 포모클럽만 **"취향 매칭 × 쉬운 번역"** 사분면에 단독.
- **토스·카카오는 개인화를 안 한다(못 하는 게 아니라 그게 그들의 모델이 아니다).** 증권사는 "객관적 시장 정보를 모두에게 동일하게" 주는 게 본질 — "지금 많이 언급되는 이슈", "올해 수익률 높은 순", "과열 신호" 전부 **전체 시장 기준 랭킹**이다. 마이크론이 뜨면 모두에게 마이크론. 개인화 0.
- 우리는 **각자 다른 화면.** 반도체에 멈칫한 사람에겐 반도체·연관주가 계속, 코인에 멈칫한 사람에겐 코인이 계속. 틴더가 스와이프로 이성 취향을 배우듯, 포모클럽은 스와이프로 투자 취향을 배워 매칭한다.
- 해자는 데이터 양 싸움이 아니라 **편집 방향 싸움**이라 저들이 안 따라온다(따라오면 자기 정체성을 버려야 함).

### ⚠️ 폐기된 프레이밍 (하지 마라)
- ❌ "감정 컨트롤 / 진정 / 위로 / 잠시 물러나라 / 안 사도 괜찮아" — **위험하다.** "물러나라" 했는데 그날 오르면 "포모클럽 때문에 놓쳤다"고 제품을 탓한다. 매매 타이밍·감정 조언은 맞히면 본전, 틀리면 원망. 너무 주관적이라 아무도 안 한 자리 = 함정이지 해자 아님.
- ❌ "종목 평가가 아니라 감정 진단" — 이것도 폐기. 우리는 감정을 진단하지 않는다.
- ✅ 우리가 하는 것: **취향대로 보여주기 + 어려운 걸 쉽게 번역.** 판단·결정은 유저 몫. 우리는 매칭과 번역만.

## 3. 핵심 가설 (재정의)

> **사용자가 (1) 스와이프로 자기 취향을 알려주고 (2) 그 취향대로 종목·테마 카드가 매칭되어 흐르고 (3) 어려운 분석(커뮤니티·시황·기술적·수급)을 쉽게 번역해주면, 매일 재방문한다.**

3가지 축: **취향 학습**(스와이프=입력) → **개인화 매칭**(각자 다른 피드) → **쉬운 번역**(어려운 걸 친구처럼).

- 매일 오는 이유(훅) = 내 취향대로 흐르는 신선한 피드. 릴스를 매일 여는 것과 같은 메커니즘.
- 차별점(해자) = **개인화 취향 매칭**. 토스가 못 하는(안 하는) 단 하나.
- 정보·번역의 질은 입장권이지 해자가 아니다. 해자는 "너만의 화면"이다.

### 콜드스타트 (반드시 인지)
새 유저는 취향 데이터가 0 → 처음엔 개인화할 게 없다. 릴스도 처음엔 아무거나 보여주다 점점 정확해진다. 그래서 **초기엔 인기·이슈 기반 피드로 시작 → 스와이프가 쌓이며 점점 개인화**되는 구조여야 한다. "첫날 텅 빈 개인화"를 만나지 않게.

## 3.5 핵심 로드맵 — 알고리즘 피드로 가는 길

제품의 본체는 취향 알고리즘 피드다. 그런데 알고리즘은 한 번에 못 만든다. **순서가 있다.**

```
[지금~Phase 4] 카드 공급 엔진 — 피드의 연료
   - 지금: 고정 5테마(반도체·AI·코인·금리·2차전지)뿐. 5장으론 알고리즘이 큐레이션할 게 없다.
   - 늘려야 함: 섹터 카드 + 개별 종목 카드 + 매크로 이슈 카드(호르무즈·AI수출통제 등).
   - "취향대로 보여주려면 보여줄 카드가 많아야 한다." 공급 풍부화가 1차 과제.
        ↓
[Phase 5] 취향 학습 — 알고리즘의 입력
   - 스와이프(관심/덜관심)·체류·검색·깊이열람을 신호로 "이 사람의 투자 취향" 프로파일 구축.
   - 흔들림 점수(§8.1)도 여기서 파생.
        ↓
[Phase 6] 개인화 피드 — 진짜 본체
   - 취향 따라 카드 순서·구성을 개인화. "릴스처럼."
   - 검색은 이 위에 얹는 보조(니즈 발동 시).
```

### 결정적 진실 (잘못 알면 헛고생)
- **개인화 알고리즘보다 "카드 공급 풍부화"가 먼저다.** 릴스 추천이 작동하는 건 볼 영상이 무한히 많아서다. 카드가 5장이면 알고리즘이 돌 무대가 없다. 알고리즘 자체보다 그 알고리즘이 큐레이션할 카드를 매일 충분히 만드는 게 더 큰 숙제.
- **초기 카드 품질은 사람(User Zero)이 잡는다.** 알고리즘·데이터가 빈약한 초기엔 4년차 트레이더이자 페르소나인 운영자가 직접 User Zero로서 카드를 보고 피드백하며 품질·취향 적합도를 잡는다. 가내수공업이 부끄러운 게 아니라 정석 — 알고리즘은 데이터(많은 카드 + 많은 사용자 행동)가 쌓여야 의미 있다.

## 4. 세 개의 화면 (스와이프 위에 얹는 레이어)

스와이프 카드 피드는 메인으로 유지. 아래 셋은 그 위/뒤/옆에 붙는다.

### 4.1 오늘의 너 (리텐션 훅) — 스와이프 끝에 뜸
카드를 다 넘기면 마지막에 뜨는 요약 한 장.
- **오늘의 흔들림 점수**(0~100) + 어제 대비 변화
- **시장 포모(집단) vs 나의 반응 대조**: "시장은 광기 84인데 너는 잘 안 흔들렸어"
- **어제→오늘 분기**(§4.4): 관심이 일관되면 "어제 그 반도체 어떻게 됐어", 튀었으면 "시선이 옮겨다니네"
- 스트릭("안 흔들린 날 5일 연속")

### 4.2 나의 포모 패턴 (해자) — 별도 탭, PRO
데이터가 쌓인 충성 사용자에게 잠금 해제(예: 2주+ 사용).
- **주간 흔들림 곡선**: 일주일을 이어 패턴을 봄. 튄 날에 사용자가 스스로 인과를 떠올림("아 그날 반도체 난리였지").
- **약한 키워드**: "너는 코인엔 단단한데 반도체엔 약해." 그 사람만의 데이터로만 만들어지는 거울 = 해자.
- **성장 지표**: "한 달 전보다 31% 덜 흔들림." 스트릭·도파민·손실회피가 여기 들어가되 방향이 반대 — **"안 한 것"을 보상**한다(안 흔들린 날 연속).

### 4.3 판단 보조 (BM 심장) — 카드 탭하면 열림
"따라가도 될까?"가 결제 트리거. 감정의 정점에서 페이월.
- **무료 = 진단**: "지금 반도체 뜨겁고, 너도 흔들리는 중이야." (불안 인지 → 훅)
- **PRO = 판단 근거 3종**(잠금):
  - **타이밍**: "이미 3일 전부터 뜨거웠어, 지금은 늦은 쪽" (객관적 기사 + 기술 지표를 쉽게 풀어서)
  - **너의 약점**: "너는 반도체에 약해, 지금이 약한 순간" (흔들림 패턴 결합)
  - **지난번의 너**: "지난달 AI도 안 따라갔잖아, 그 뒤 이렇게 됐어" (과거 데이터로 판단 근거)
- **절대 규칙**: 답("사라/마라")을 주지 않는다. 판단을 사용자에게 돌려준다. "늦은 쪽에 가까워"지 "들어가지 마"가 아니다. 면책("결정은 너의 몫")은 장식이 아니라 방패.

### 4.4 어제→오늘 분기 로직 (스키마가 안 무너지게)
**추적 단위를 "키워드"가 아니라 "사용자의 상태"로 올린다.** 관심이 매일 어디로 튀든 스키마가 항상 채워진다.
- 어제와 오늘 관심이 **일관** → "어제 그 반도체, 오늘 이렇게 됐어"(연속 추적)
- 어제와 오늘이 **튐** → "어제는 반도체, 오늘은 코인이네. 시선이 옮겨다니는 것도 포모야"(흔들림 감지)
- **핵심 통찰**: 관심이 튀는 것 자체가 포모의 증거다. 무너지는 스키마가 아니라 흔들림 점수의 입력이다.

## 5. BM (조언을 팔지 않는다는 전제 위에서)

내러티브: **무료 일일 안티-포모 피드 → 습관 + 감정적 신뢰 + 관심 데이터 → 두 자산이 복리로: (1) 관계, (2) 지수.**

| 레이어 | 무료 | PRO (구독) |
|---|---|---|
| 키워드 카드 | 오늘의 키워드 + 점수 + 친구톤 코멘트 | — |
| 용어 풀이 | 펀더멘탈·EPS를 쉽고 **따뜻하게** (불안 안 키우는 게 기준) | — |
| 오늘의 너 | 흔들림 점수 + 어제 대비 | — |
| 판단 보조 | 진단("흔들리는 중") | 판단 근거 3종(타이밍·약점·과거의 나) |
| 패턴 거울 | — | 주간 곡선 + 약한 키워드 + 성장 |

- **결제 타이밍 = 감정의 정점.** "따라가고 싶어 미치겠는데 맞는지 모르겠는" 순간에 게이트. 포모 자체가 결제 트리거. 불안 증폭이 아니라 **불안 해소의 대가**라 정체성과 안 부딪침.
- **PRO의 깊은 가치는 충성 사용자만 느낀다.** 약점·과거의 나는 데이터가 쌓여야 의미 있음 → 신규는 잠겨도 안 아깝고, 오래 쓸수록 잠긴 게 탐남 = 자연스러운 충성 게이트.
- (장기) FOMO Index 라이선싱 / (서랍) 타로 부활은 PRODUCT_TRUTH §7 참조.

## 6. 유통 전략 (SNS → 앱, 웹은 검증장 겸 랜딩)

- **장기 그림**: 매일 여는 건 앱. SNS 카드레터로 트래픽 → 앱 설치. 웹은 랜딩.
- **단, 지금은 웹 우선.** 검증 안 된 제품을 설치 마찰 뒤에 숨기면 "설치를 안 해서인지 제품이 별로인지" 구분 불가. 웹(설치 마찰 0)으로 **"매일 열 만한가"를 먼저 검증** → 증명되면 앱으로 옮기며 푸시·개인화로 리텐션 증폭.
- **시점**: 웹이 D7 재방문을 검증한 뒤 앱 착수. 그 전까지 앱(RN)은 보류(PRODUCT_TRUTH §2).
- 카드레터는 지금부터 웹으로 트래픽 보내 "다시 오는가" 측정 → 앱 출시 시 설치 전환.

## 7. Phase 1~4가 미리 깔아둘 것 (Phase 5 되돌아가기 방지)

엔진을 만들 때 아래를 염두에 둔다. 지금 구현하라는 게 아니라, **나중에 알고리즘 피드·흔들림 점수가 쓸 데이터를 버리지 말라**는 뜻.

- **카드 공급 확장 가능하게(최우선)**: 지금은 고정 5테마. 추후 섹터·개별 종목·매크로 이슈 카드로 늘어난다(§3.5). 키워드/카드 타입을 5테마에 하드코딩하지 말고, 카드 종류가 늘어도 깨지지 않게 추출·점수·카드 스키마를 일반화해 둘 것. (알고리즘이 큐레이션할 "연료"가 많아야 피드가 성립)
- **키워드 점수에 "언제 시작됐나"(쏠림 시작 시점) 기록**: §4.3 판단 보조의 "타이밍"이 이걸 쓴다. 일일 스냅샷에 키워드별 첫 감지일·연속일 보존.
- **사용자 행동 로그 보존(취향 신호)**: 스와이프(관심/덜관심), 카드 탭, 본 키워드, 머문 시간. 이게 곧 §3.5 취향 학습의 입력이자 흔들림 점수의 재료. 이미 `keywordHistory`가 로컬 저장 중 → 서버 집계로 확장.
- **익명 세션 단위 일관성**: 흔들림 점수·취향 프로파일은 세션별 시계열이라 세션 ID가 안정적으로 유지돼야 함.
- **confidence 보존**: 흔들림 점수도 데이터 적으면 정직하게 폴백(콜드스타트, §8).

## 8. 미해결 숙제 (Phase 5 착수 전 풀 것)

다음 둘이 풀려야 §4 세 화면이 진짜가 된다. 안 풀면 또 가짜 숫자.

### 8.1 흔들림 점수 정의
**무엇으로 0~100을 만드나?** 후보 신호: 시장 광기 키워드와 내 관심의 겹침 / 스와이프 속도·양 / 평소 대비 검색·탭 증가 / 관심이 튄 정도(§4.4). 공식·가중치·정직성(가짜 점수 금지)을 별도 설계 필요.

### 8.2 콜드스타트
첫날은 비교할 어제가 없다. 첫 3일은 "너를 알아가는 중" 온보딩으로 프레이밍. 약점 분석은 2주+ 데이터 후 잠금 해제. 데이터 부족 시 점수 대신 정직한 빈 상태.

## 9. 개발 순서 & 매뉴얼

### 전체 순서
```
[지금] Phase 1 → 2 → 3 → 4 (엔진을 진짜로) — KEYWORD_ENGINE_SPEC §6
          ↓ (각 Phase 끝나면 멈추고 검증, Phase 2 후엔 실제 출력 같이 확인)
[검증] 웹에서 D7 재방문 측정 (PRODUCT_TRUTH §5 지표)
          ↓ (매일 열 만한가? 증명되면)
[Phase 5] 흔들림 점수 정의(§8.1) → 오늘의 너(§4.1) → 판단 보조 무료 진단(§4.3 무료)
          → 패턴 거울 + PRO 판단 근거(§4.2, §4.3 PRO) → 구독 결제
          ↓
[그 후] 앱(RN) 착수 + 푸시·개인화 증폭 → 카테고리 확장
```

### Phase 5 내부 순서 (엔진 검증 후)
1. **흔들림 점수 엔진**(§8.1 정의 → 순수 함수 + 테스트). 세 화면의 공통 토대라 먼저.
2. **오늘의 너**(§4.1): 점수를 스와이프 끝 카드로 노출. 어제→오늘 분기(§4.4) 포함. 무료.
3. **판단 보조 — 무료 진단**(§4.3 무료): 카드 탭 시 "흔들리는 중" 진단.
4. **패턴 거울 + PRO 게이트**(§4.2): 주간 곡선 + 약한 키워드. 2주+ 사용자 잠금 해제.
5. **PRO 판단 근거 + 결제**(§4.3 PRO): 타이밍·약점·과거의 나 + 구독.

### 운영 매뉴얼 (Phase 트랙 공통)
- **한 번에 한 Phase만.** 끝나면 Acceptance 결과 + 변경 파일 보여주고 멈춤. 다음은 직접 지시.
- **Phase별 PR로 끊어 머지.** rollback-friendly. 큰 변경 섞지 말 것.
- **민감 작업은 게이트.** prod DDL(`db push`), 결제 연동 등은 자동 실행 금지, 직접 승인.
- **정직한 숫자 원칙은 Phase 5에도 그대로.** 흔들림 점수가 임의로 나오면 즉시 폐기. confidence 노출.
- **에이전트 자율 루프는 정지 유지.** 엔진+Phase 5 안정 후 재개. 재개 전 제안 스크립트가 PRODUCT_TRUTH·PRODUCT_VISION을 읽는지 확인.
- **스와이프 UX는 건드리지 않는다.** 모든 신규 화면은 스와이프 위에 얹는 레이어.

### 검증 게이트 (Phase 5 착수 조건)
Phase 1~4 끝 + 웹에서 D7 재방문 기준선 확보 + "매일 열 만한가"에 YES. 이게 NO면 Phase 5 기능보다 엔진/콘텐츠 품질을 먼저 고친다. **기능을 더 얹기 전에 토대가 재방문을 만드는지부터 확인.**
