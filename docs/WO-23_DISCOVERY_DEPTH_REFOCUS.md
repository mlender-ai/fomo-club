# WO-23: 발견 중심 뎁스·차트 재정렬

## SPEC

### 1. 문제
- 종목 뎁스 첫 화면을 차트가 점유해 FOMO Club이 분석 도구처럼 보인다.
- `왜 움직였나`가 근거 없는 고정 5거래일을 강조해 사건과 기간의 관계가 불명확하다.
- 주식용 verdict가 코인 카드에서 대부분 `관망`으로 수렴해 서로 다른 시장 상태가 구분되지 않는다.
- 코인 뎁스도 주식 재무·인사이트 요청을 기다려 첫 렌더가 늦다.

### 2. 목표
- 한 줄 목표: 차트는 읽기 쉽게 만들되, 뎁스의 첫 답은 "오늘 왜 발견했는가"가 되게 한다.
- 카드에서 주목 상태와 차트 균형을 별도 축으로 표현한다.
- 고정 5일 비교를 없애고 사용자가 선택한 1개월·3개월 구간 변화를 표시한다.
- 코인 뎁스는 코인 front만으로 즉시 열리게 한다.
- 코인·미장 프리웜 완료 시 12시간 덱과 10분 종목 뎁스 캐시를 무효화해 빈 자산군·차트가 고정되지 않게 한다.

### 3. 성공 지표
- 카드의 `진입 검토/관망/회피` 문구 0건.
- 코인 카드가 실제 포모 상태에 따라 `관심 붙는 중/조용/열기 식는 중` 등으로 구분된다.
- 뎁스 첫 화면에 `오늘 발견 포인트`가 차트보다 먼저 노출된다.
- 차트에 기간, OHLC, 가격축, 거래량 구획이 보이고 5D 문구는 0건이다.
- PC·390px 모바일에서 텍스트·차트 겹침이 없다.

### 4. 절대 제약
- `docs/PRODUCT_VISION.md`의 발견 중심, TA 보조 사실 원칙을 지킨다.
- 매수·매도·목표가·예측 문구를 만들지 않는다.
- 새 데이터 소스와 추정 인과를 추가하지 않는다.
- 기존 사용자 작업과 피드·스토리 플로우를 훼손하지 않는다.

### 5. 비범위
- 거래 주문, 호가창, 실시간 틱, TradingView 복제.
- 코인별 뉴스·온체인 신규 소스.
- verdict 엔진 자체의 임계값 재튜닝.

## PLAN

1. `discoveryPresentation` 순수 함수로 포모 상태와 차트 균형 표현을 분리한다.
2. PC 목록과 모바일 스와이프 카드에서 매매형 stance 라벨을 발견 상태로 교체한다.
3. 종목 뎁스 상단에 기존 reason·source·fomo·volume을 응축한 발견 요약을 둔다.
4. 반응 차트의 5일 강조를 제거하고 선택 구간 OHLC·변화율·가격축을 추가한다.
5. 구조 차트는 1·3·6개월 창과 가격축을 제공해 캔들 밀도를 낮춘다.
6. 코인 뎁스는 재무·주식 인사이트 요청을 건너뛴다.
7. 코인·미장 프리웜이 `daily-30`·`feed-hub`와 자산군별 `stock-front`를 즉시 재검증하게 한다.
